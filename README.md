# 欢乐斗地主（单机版 + 联机版）
本项目是一款采用 C++11 面向对象架构、基于 Qt 框架开发的斗地主游戏，包含单机对战和网络联机两个版本，实现了经典斗地主规则、AI 对战、自定义界面、音效及多人联机功能。

## 项目特性
### 单机版
- 完整游戏流程：发牌、叫地主、出牌、结算
- AI 自动叫牌、出牌
- 自定义卡牌显示、牌型动画
- 背景音乐、操作音效、语音提示
- 鼠标点击选牌、出牌
- 实时计分

### 联机版
- 支持 3 人网络联机对战
- 服务端管理房间、玩家状态与游戏逻辑
- 客户端与服务端通过 JSON 协议通信
- 支持心跳包保持连接
- 实时同步玩家出牌、叫地主与分数信息

## 系统架构设计

### 单机版架构
项目采用四层结构，模块职责清晰，便于维护与扩展：

#### 表现层（Presentation Layer）
负责界面显示、用户操作、动画与音效。

| 核心模块 | 核心职责 |
|----------|----------|
| GamePanel | 游戏主窗口，管理界面与游戏流程 |
| CardPanel | 卡牌显示与点击交互 |
| AnimationWindow | 牌型动画播放 |
| CountDown | 出牌倒计时 |
| EndingPanel/ScorePanel | 结算界面与分数显示 |
| ButtonGroup/MyButton | 自定义按钮 |
| Loading | 启动加载界面 |

#### 业务逻辑层（Business Logic Layer）
处理游戏规则与流程控制。

| 核心模块 | 核心职责 |
|----------|----------|
| GameControl | 控制游戏状态：发牌→叫地主→出牌→结算 |
| PlayHand | 识别牌型、判断大小、校验合法性 |
| Strategy | AI 叫分与出牌逻辑 |
| Player/UserPlayer/Robot | 玩家与 AI 对象 |

#### 数据模型层（Data Model Layer）
卡牌与玩家数据结构定义。

| 核心模块 | 核心职责 |
|----------|----------|
| Card | 单张卡牌定义 |
| Cards | 一组卡牌的集合操作 |
| 玩家数据模型 | 记录角色、分数、手牌等信息 |

#### 基础设施层（Infrastructure Layer）
提供底层功能支持。

| 核心模块 | 核心职责 |
|----------|----------|
| BGMControl | 音频播放管理 |
| Qt资源系统 | 资源打包与加载 |
| 多线程基础设施 | AI 异步运行 |

采用客户端-服务端（C/S）架构，服务端负责游戏逻辑与玩家管理，客户端负责界面展示与用户交互。

## 核心技术实现

### 单机版核心实现

#### 1. 游戏流程控制
使用状态机管理游戏流程：
- 发牌：随机洗牌，分发手牌，预留3张底牌
- 叫地主：三人轮流叫分，1~3分，分数高者为地主
- 出牌：轮流出牌，可压牌、不出，先出完者获胜
- 结算：计算分数，更新玩家得分

#### 2. 牌型识别与规则
支持所有标准斗地主牌型：
单张、对子、三张、三带一、三带二、顺子、连对、飞机、飞机带翅膀、炸弹、王炸等。
自动判断牌型是否合法、能否压制上家牌型。

#### 3. AI 决策系统
##### 叫地主评分规则
根据手牌评分决定叫分：
- 大小王：每张6分
- 炸弹：每个5分
- 顺子：每组5分
- 2：每张3分
- 三张：每组4分
- 对子：每组1分

评分对应策略：
- ≥22 分：叫3分
- 18~22 分：叫2分
- 10~18 分：叫1分
- <10 分：不叫

##### 出牌策略
- 优先出小牌，保留大牌
- 能压制则压制，不能则不出
- 残局优先防止对手跑牌
- 不随意拆炸弹、王炸

##### 多线程运行
AI 思考在独立线程运行，模拟思考延迟，界面保持流畅。

#### 4. 界面与动画
- 自定义绘制卡牌
- 支持选中、移动、出牌动画
- 炸弹、王炸等特效显示
- 界面风格统一，操作简单

#### 5. 音频播放
支持5类音频同时播放，互不冲突：
- 男/女语音
- 背景音乐
- 操作音效
- 结算音乐

### 联机版核心实现

#### 1. 服务端模块
##### Server 模块
- 监听指定端口，接受客户端连接
- 管理玩家认证（验证玩家名字与性别）
- 广播消息给所有客户端
- 处理客户端断开连接，清理玩家数据

##### GameMaster 模块
- 在独立子线程运行，避免阻塞服务端界面
- 管理游戏初始化、发牌、流程控制
- 接收 Server 转发的客户端消息并处理
- 协调 GameControl 与 Server 的数据交互

##### GameControl 模块（服务端）
- 初始化与洗牌
- 分配玩家手牌与地主底牌
- 处理叫地主逻辑，判断谁成为地主
- 处理出牌逻辑，校验牌型，判断胜负
- 计算玩家分数，触发结算

#### 2. 客户端网络模块
在单机版 `GamePanel` 基础上扩展：
- 使用 `QTcpSocket` 连接服务端
- 发送连接认证消息（玩家名字、性别）
- 发送心跳包保持连接
- 接收服务端消息，使用 Qt JSON 模块解析
- 根据服务端指令更新界面（发牌、叫地主、出牌、结算）

#### 3. 通信协议设计
使用 JSON 格式进行数据传输，消息前加 4 字节大端序长度字段处理粘包。

##### 典型消息示例
- **连接认证**
```json
{
  "cmd": "connect",
  "data": {
    "player_name": "张三",
    "player_sex": 1
  }
}
```
- **发牌**
```json
{
  "cmd": "deal_cards",
  "data": {
    "players": [
      {
        "player_id": "张三",
        "cards": ["0_3", "1_5", "2_7"]
      }
    ],
    "landlord_cards": ["3_13", "3_14", "0_2"],
    "nextcall": "张三"
  }
}
```
- **叫地主**
```json
{
  "cmd": "call_lord",
  "data": {
    "player_id": "张三",
    "point": 3,
    "nextcall": "李四"
  }
}
```
- **出牌**
```json
{
  "cmd": "play_hand",
  "data": {
    "player_id": "张三",
    "cards": ["0_3", "0_4", "0_5"],
    "gameover": false,
    "nextplayhand": "李四",
    "timeout": false
  }
}
```
- **出牌**
```json
{
  "cmd": "game_over",
  "data": {
    "name_1": "张三",
    "score_1": 10,
    "name_2": "李四",
    "score_2": -5,
    "name_3": "王五",
    "score_3": -5
  }
}
```

## 编译与部署指南
### 环境要求
- Qt 5.15.2 / Qt 6.5.0+
- Windows：MSVC 2019/MinGW-W64 8.1.0
- Linux：Ubuntu 20.04，GCC 9.4.0
- macOS：macOS 12+，Clang 13.0+

### 1. Qt Creator 构建（推荐）
1. 打开 Qt Creator，导入项目 .pro 文件
2. 配置编译环境
3. 切换到 Release 模式
4. 构建并运行

### 2. 命令行构建（qmake）
```bash
cd HappyLandlords
qmake HappyLandlords.pro
```
Windows MinGW：
```bash
mingw32-make -j8
release/HappyLandlords.exe
```
Windows MSVC：
```bash
nmake
release/HappyLandlords.exe
```
Linux/macOS：
```bash
make -j8
./release/HappyLandlords
```
### 3. 部署发布
1. Windows
```bash
windeployqt --release --compiler-runtime HappyLandlords.exe
```
- 将 resource.rcc 放在同一目录即可分发。
2. Linux
- 使用 ldd 查看依赖库，复制依赖库与程序同目录，搭配 resource.rcc 发布。
3. macOS 
```bash
macdeployqt HappyLandlords.app
```
- 将 resource.rcc 放入 app 包内 Resources 目录。

## 联机版编译与部署
1. 使用 Qt Creator 打开服务端项目 .pro 文件
2. 配置编译环境，切换到 Release 模式
3. 构建生成可执行文件
4. 运行服务端，在界面输入端口号（如 8080），点击 “启动”
## 客户端编译与部署
1. 在单机版代码基础上，确保包含网络模块（在 .pro 文件中添加 QT += network）
2. 编译生成可执行文件
3. 部署方式同单机版
4. 运行客户端，输入服务端 IP 地址与端口号，连接后输入玩家名字即可开始游戏

## 个人总结
跟着课程敲完单机版的代码，我才算真正吃透了 C++ 面向对象、多态这些核心特性，也把 Qt 的信号槽机制、常用的设计模式练熟了，扎扎实实打下了编程基础。
本来想着顺着单机版的底子做联机版会轻松些，结果没了课程跟着敲，才发现这件事对当时的我来说有多难。为了实现联机功能，我硬着头皮啃了好多网络编程的内容：最基础的 Socket 套接字编程、粘包问题的处理方案、心跳包的保活机制，还有 IO 多路复用 —— 好在 Qt 已经把这块做了封装，省了我大量的功夫，也让开发效率提上来了。还有 JSON 数据的打包和解析，在 Qt 里做起来确实很便利，不过后面我也打算抛开 Qt 的封装，系统地把网络编程这块知识再深耕一遍。
整个联机版开发下来，最头疼的就是客户端和服务端的信息对接，还有怎么把网络逻辑和单机版已经写好的 API 接口适配起来，中间踩了不少坑。不过好在最后还是完整把项目做出来了，真的收获满满，项目跑通的那一刻，成就感直接拉满。
